# Gemini CLI セキュリティガイド

Gemini CLI は、ホストシステムから潜在的に危険な操作（シェルコマンドやファイルの変更など）を隔離するためにサンドボックスを使用します。このガイドでは、悪意のあるスクリプトや拡張機能による不正な実行に対する保護機能を有効にする方法について説明します。

## 脅威

攻撃者は、悪意のある VS Code 拡張機能やスクリプトを使用して、Gemini CLI を密かに呼び出し、データを外部に送信する可能性があります。Socket.dev のレポートでは、以下のエクスプロイトコマンドが特定されています。
`gemini prompt "prompt" --yolo --no-stream`

このコマンドは、人間による承認なしに実行するようにエージェントに指示し、システム全体の機密データを読み取って攻撃者のサーバーに送り返す可能性があります。

## 解決策

Gemini CLI のサンドボックスを設定することで、OS レベル（macOS の Seatbelt）またはコンテナ化（Docker/Podman）によって強制される厳格な境界を定義します。

### 推奨設定 (`settings.json`)

提供されている [`settings.json`](settings.json) を設定ファイルのパスにコピーしてください。

主要なセキュリティ設定は以下の通りです。

- **`"sandbox": true`**: サンドボックスランタイムをアクティブにします。Gemini CLI によって実行されるすべての操作はサンドボックスによって制限され、プロジェクトディレクトリ外への無制限の書き込みが防止されます。

あるいは、環境変数（`GEMINI_SANDBOX=true` または `GEMINI_SANDBOX=docker`）または `-s` コマンドフラグを使用してサンドボックスを有効にすることもできます。

## きめ細かなセキュリティフック

サンドボックスは任意のファイルの書き込みを防ぎますが、攻撃者がすでに認証済みの CLI ツール（`gh` や `curl` など）を使用してサンドボックス内からデータを外部に送信することを防ぐことはできません。

これに対処するために、コマンドがシェルによって実行される前にそれらをインターセプトする **BeforeTool フック**を使用します。

### `gh-safeguard.sh` フックの使用方法

1. `hooks/gh-safeguard.sh` スクリプトをプロジェクトの `.gemini/hooks/` ディレクトリにコピーします。
2. スクリプトを実行可能にします: `chmod +x .gemini/hooks/gh-safeguard.sh`
3. `settings.json` に、提供されている例の `hooks` 設定が含まれていることを確認してください。これにより、シェル実行ツールの前にこのスクリプトが実行されるように登録されます。

このスクリプトは LLM が実行しようとしているコマンドを解析し、`gh repo create` や `gh gist create` のような既知のデータ外部流出コマンドを厳格にブロック（終了コード 2）し、安全なクエリコマンドの続行を許可します。

**リポジトリの固定（Repository Pinning）**: このフックは、`-R` および `--repo` フラグ（および `repos/owner/repo` のような明示的な API パス）もブロックします。これにより、`gh` CLI は強制的にローカルワークスペースのコンテキストを使用することになり、エージェントを現在のリポジトリに効果的に「固定」し、攻撃者が制御する外部リポジトリにデータをプッシュするのを防ぎます。

## 検証

Gemini CLI が正しく設定され、`gh-safeguard.sh` フックがアクティブであることを確認するために、リポジトリの統合テストスイートを使用できます。

```bash
make test-integration
```

これにより、自動化されたプローブが実行され、`gemini prompt 'gh repo create'` のようなコマンドが正常にブロックされることが確認されます。

## サンドボックスモード

プラットフォームとセットアップに応じて、Gemini CLI はさまざまなサンドボックス方式を使用します。

- **macOS Seatbelt**: デフォルトのプロファイル（`permissive-open`）は、プロジェクトディレクトリ外への書き込みを制限しますが、他のほとんどの操作は許可します。
- **Docker/Podman**: 完全なプロセス隔離を備えたクロスプラットフォームのサンドボックス。コンテナベースの隔離を好む場合は、`GEMINI_SANDBOX=docker` を使用してください。

サンドボックスが有効な場合でも、GUI アプリケーションが正常に動作しなかったり、使用されるサンドボックスプロファイルによってはネットワークトラフィックが許可されたままになったりする可能性があります。
